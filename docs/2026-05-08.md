# 📅 2026-05-08 HTML_table-input_tag
### 🏷️ Topic
HTML의 table 태그와 form태그(input태그)

### 📝 What I Learned
구조적인 표(table)을 만들고 input 태그를 통해 다양한 기능들을 실행 시켜보았다.

table 태그

```html
<table> : <table> 전체 표를 감싸는 베이스 보드
  <thead> : <thead> 표의 상단 제목(헤더) 영역을 그룹화
    <tr> : <tr> 데이터를 담을 가로 줄 table row
      <th>구분</th> : <th> 굵은 글씨와 중앙 정렬이 기본인 제목 셀 table head
      <th>항목</th>
    </tr>
    <tr>
      <th>(대분류)</th>
      <th>(소분류)</th>
    </tr>
  </thead>
  <tbody> : <tbody> 표의 실제 데이터 본문 영역을 그룹화
    <tr>
      <td></td> : <td> 실제 데이터가 들어가는 일반 셀 table data
      <td></td>
    </tr>
  </tbody>
  <tfoot> : <tfoot> 표의 하단 요약이나 합계 영역을 그룹화
    <tr>
      <td></td>
      <td></td>
    </tr>
  </tfoot>
</table>

<colspan> : colspan='숫자' 가로 열 합치기
<rowspan> : rowspan='숫자' 세로 열 합치기 

```

```html
웹서버에서 오류

		404	: 파일명이 틀린경우	
		500 : 자바와 연동 => 자바오류
		403 : 접근거부
		412 / 415 / 400 => JSP와 연동
```

사용자 입력도구
```html
텍스트 : 일반 한 줄 입력
type="text"

보안 : 마스킹 처리된 비밀번호
type="password"

데이터 특정 : 이메일 양식 검증
type="email"

데이터 특정 : 숫자만 허용
type="number"

특수 UI 호출 : 시스템 달력 호출
type="date"

특수 UI 호출 : 색상 팔레트 호출
type="color"

상호작용 : OS탐색기 열기
type="file"

상호작용 : 폼 최종 제출
type="submit"
```

### 💡 Note
구조를 파악하면 외우지 않아도 자연스럽게 사용할 수 있다
가끔 태그를 사용할 때 열린태그/닫힌태그인지 독립태그인지 헷가릴때가 있다.
그럴때 마다 판단하는 기준은 정해보았다.
```html
1. 무엇인가를 감싸고 있는가? y=닫아, n=독립
2. 속성만으로 충분한가? y=독립, n=닫아
```

일단 기본적으로 이번에 찾아보면서 느낀건데 독립태그가 생각보다 많지 않다. 

```html
<br>	줄 바꿈 (Enter)
<hr>	가로 구분선
<img>	이미지 삽입
<input>	입력 필드 (글자 입력, 버튼 등)
<meta>	문서 정보 (Head 태그 안)
<link>	외부 파일 연결 (CSS 등)
```
이정도는 알아두자
